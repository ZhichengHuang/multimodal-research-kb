## 一句话总结
用 Empirical Bayes 收缩估计替代 GRPO 的纯局部 group 统计量作为 baseline，解决饱和失败场景下的梯度消失问题并降低 baseline 估计方差，在小 group size (G=4/8) 下显著优于 GRPO。

## 核心 Insight
**GRPO 的纯局部 baseline 是信息浪费** — 当一个 prompt 的所有采样都失败时（saturated failure），局部 group 统计量给出零方差、零梯度，完全丧失学习信号。EBPO 通过将局部均值与全局策略成功率动态插值（shrinkage），在这种场景下仍能产生有信息量的负梯度信号，且失败惩罚随任务全局难度自适应缩放。

## 与已有工作的关系
- **继承自**: [[DeepSeek-R1]]（GRPO 算法，EBPO 直接改进其 baseline 估计机制；advantage 计算的 group-based 框架和 batch 内标准化均继承自 GRPO）、James-Stein Shrinkage Estimator（经典统计学，收缩因子 S_q 直接源自 James-Stein 类估计量的构造）
- **对比**:
  - [[2026-LaViDa-R1]]: **解决同一问题（训练信号消失）的两种正交方案**。EBPO 通过全局先验注入（shrinkage baseline V^EB = S_q * mu_glob > 0）产生负 advantage 信号（"这个方向不行"），LaViDa-R1 通过 answer-forcing（利用 dLLM inpainting 注入 ground-truth answer）产生正向训练样本（"这条推理路径可行"）。EBPO 是架构无关的统计方法，answer-forcing 是 dLLM 架构专有能力。两者正交可组合
  - [[2025-OpenMMReasoner]]: 同为改进 GRPO 的工作。GSPO 的具体改进机制未详细公开，EBPO 的 shrinkage baseline 有清晰统计理论支撑（James-Stein MSE 严格降低）。GSPO 在传统 VLM（Qwen2.5-VL-7B）上验证，EBPO 在纯语言模型（Qwen3-8B/14B）上验证。两者改进可能正交——GSPO 的 group 构造改进 + EBPO 的 baseline 改进可叠加
  - DAPO / Dr.GRPO / EntropyMech: 直接竞争方法。EBPO 在 Qwen3-8B 上全面领先（64.39% vs DAPO 52.63%, Dr.GRPO 56.67%, EntropyMech 49.18%）。各方法的改进维度（DAPO clip-higher/dynamic sampling, Dr.GRPO 梯度修正, EntropyMech entropy 正则化）与 EBPO shrinkage baseline 正交，理论上可叠加组合
- **互补**:
  - [[2025-MMaDA]]: EBPO 的 shrinkage baseline 可直接替换 MMaDA UniGRPO 中的 GRPO baseline 估计。MMaDA 明确"训练信号消失问题未解决"，EBPO 直接缓解。dLLM 图像生成 rollout 成本高→group size 受限，正是 EBPO 优势最大的区间
  - [[2025-Lumina-DiMOO]]: EBPO 可替换 DiMOO Self-GRPO 的 baseline 估计。Self-GRPO 的自评估 reward 有 bootstrapping 偏差→噪声较高，EBPO 的方差降低在噪声 reward 场景下价值更大。EBPO 的 topic-coherent sampling 可推广到 DiMOO 多任务 RL（按任务类型聚类）
  - [[2025-MMaDA-Parallel]]: EBPO 可改进 ParaRL 的 advantage 估计。ParaRL 的轨迹级 CLIP reward 在中间步骤噪声大，EBPO 方差降低对此特别有价值。ParaRL 未解决训练信号消失，EBPO 部分缓解
  - [[2025-LaViDa]]: Complementary Masking（降低 likelihood 估计方差）+ EBPO shrinkage（降低 advantage baseline 方差）作用于 RL 管线不同环节，可叠加实现双重方差降低

## 方法细节

### 问题: GRPO 的两个关键缺陷

GRPO 计算 advantage 的方式:
- Â_i = (r_i - μ_group) / (σ_group + ε)
- 其中 μ_group = (1/G)Σr_j, σ_group 为组内标准差

**缺陷 1: 饱和失败时梯度消失** — 当 group 内所有采样的 reward 均为 0 时（常见于困难数学题 + 小 group size），μ_group=0, σ_group=0，所有 advantage 为 0，策略得不到任何更新信号。

**缺陷 2: 小 group size 下的高方差** — G=4/8 时，局部均值对真实 prompt 难度的估计方差为 σ²/G，噪声极大。

### EBPO 核心设计

**收缩 Baseline:**
- V^EB_q = (1 - S_q) × μ_group + S_q × μ_glob

**收缩因子:**
- S_q = (σ²/G) / (σ²/G + τ²)
  - σ² = 组内方差（单个 reward 的方差）
  - τ² = 组间方差（prompt 均值的方差）
  - μ_glob = 全局策略成功率

**直觉:** 当组内估计噪声大（σ²/G 大）或 prompt 间难度差异小（τ² 小）时，S_q → 1，更依赖全局先验；当组内估计精确或 prompt 间差异大时，S_q → 0，更信任局部估计。

**最终 Advantage:**
- Â_m,i = [(r_{m,i} - V^EB_qm) - μ_Â] / (σ_Â + ε)
  - 在 batch 内再做标准化

**在线统计维护:** 使用 Welford's Online Algorithm 维护 μ_glob, σ², τ² 的运行估计，避免小 batch 统计噪声。

## Building Blocks（可复用组件）

### Block 1: Empirical Bayes Shrinkage Baseline
- **做法**: 将 GRPO 的纯局部 group mean 替换为局部-全局插值 baseline，插值权重由组内/组间方差比自动决定
- **机制 (WHY it works)**:
  - James-Stein 类收缩估计在统计学中被证明对同时估计多个均值时严格优于 sample mean（MSE 更低）
  - 收缩将极端估计（如全 0）"拉"向全局先验，减少噪声对梯度的影响
  - 在 saturated failure 下，V^EB = S_q × μ_glob > 0，产生负 advantage（惩罚失败），而非零梯度
  - MSE_EBPO = (1-S_q) × σ²/G < MSE_GRPO = σ²/G（严格降低）
- **适用条件**:
  - RL 训练使用 group-based advantage 估计（GRPO/DAPO 系列）
  - 训练数据中存在大量困难样本（高 saturated failure 率）
  - Group size 受限（G=4/8），局部统计量方差大
- **什么时候会 break**:
  - Group size 足够大（G≥32）时，局部统计量已足够精确，收缩的边际收益递减
  - 全局先验的偏差系统性地误导（如训练分布非稳态变化太快，Welford 估计滞后）
  - Gaussian 近似假设不成立（θ_q 的分布严重偏斜时收缩可能不优）
- **可组合方向**:
  - 可直接替换 KB 中所有使用 GRPO 的工作的 baseline 估计（如 [[2025-OpenMMReasoner]] 的 GSPO、[[2025-MMaDA]] 的 UniGRPO、[[2025-Lumina-DiMOO]] 的 Self-GRPO）
  - 可与 dLLM RL 的 complementary masking likelihood 估计结合
  - 可与 DAPO 的 clip-higher/dynamic sampling 等技术正交组合

### Block 2: Topic-Coherent Clustered Sampling
- **做法**: 将训练 prompt 按数学领域（代数、几何、概率等 9 类）聚类，每个 mini-batch 内 prompt 来自同一主题，使 Welford 全局先验在局部窗口内更准确
- **机制 (WHY it works)**:
  - Proposition 3.8: J_shuffle = J_coherent + Var_k(μ_k)，随机 shuffle 引入的额外方差 = 主题间均值方差
  - 同主题 prompt 的 latent difficulty 分布更集中（τ²_within-topic < τ²_global），Welford 估计更准
  - 减少 Welford 在线估计器在异质分布间震荡的问题
- **适用条件**:
  - 训练数据有可识别的聚类结构（如数学领域、任务类型）
  - 可以获得 prompt 的主题/难度标签（论文用 GPT-4.1 标注）
- **什么时候会 break**:
  - 主题边界模糊，聚类质量差
  - 过度聚类导致模型在某些主题上过拟合、泛化能力下降
  - 需要均匀覆盖所有主题的场景（如 curriculum 要求交替出现不同难度）
- **可组合方向**:
  - 可推广到多模态 RL：按视觉任务类型（图表理解、空间推理、OCR 等）聚类
  - EBPO-diff（难度排序）提供另一种聚类维度：从易到难的课程学习
  - 两种聚类维度可正交组合（主题内按难度排序）

### Block 3: Difficulty-Based Curriculum Ordering (EBPO-diff)
- **做法**: 用基座模型 pass rate 估计题目难度，训练时从易到难排列
- **机制 (WHY it works)**:
  - 从易到难的课程让全局先验在训练早期更准确（简单题成功率高，μ_glob 稳定）
  - 模型在简单题上建立推理基础后，更有能力处理困难题
  - 与 EBPO 的全局先验协同——课程顺序让 Welford 估计更加平滑
- **适用条件**:
  - 可以获得题目难度的合理估计（如基座模型 pass rate）
  - 训练时间足够长，从易到难的课程有足够的"简单题暖启动"
- **什么时候会 break**:
  - 难度估计不准确时课程顺序反而有害
  - 过度集中在简单题导致模型"忘记"困难题的泛化能力
  - 在线 RL 场景中不可能预先排序
- **可组合方向**:
  - 与 [[2025-OpenMMReasoner]] 的 SFT 阶段课程学习结合（SFT 先简单后复杂 + RL 阶段 EBPO-diff）
  - 可推广到多模态场景：按图像复杂度/推理步数排序

## Anti-patterns / 已知失败模式
- **GRPO 在小 group size 下的 saturated failure**: G=4/8 时高难度 prompt 几乎必然导致全零 reward（P(全零) = 0.9^4 = 65.6%），梯度完全消失。EBPO 通过全局先验注入信号解决此问题
- **随机 shuffle 破坏先验质量**: 随机打乱 prompt 导致 Welford 在线估计器在异质分布间震荡（J_shuffle > J_coherent），应使用主题或难度聚类
- **GRPO 的隐式 entropy 崩溃**: GRPO 在饱和失败场景下虽然梯度为零（不更新策略），但在非零 advantage 场景中可能产生过大的 KL divergence spike，导致 entropy 急剧下降、策略过早收敛
- **Stale prior collapse（EBPO 自身风险）**: 训练早期策略快速提升时，Welford 的 mu_glob 滞后于当前真实成功率，系统性偏低。这导致 saturated failure 的惩罚信号 (-S_q × mu_glob) 偏弱，恰恰在最需要学习信号时信号不足。缓解: 用 EWMA 替代全历史 Welford
- **Batch normalization 抵消 shrinkage 效果**: 在同主题高难度 batch 中，大部分 prompt 都是 saturated failure，EBPO 赋予的负 advantage 相似。Batch normalization (减均值/除标准差) 将这些负信号映射回零附近，部分抵消 Theorem 3.2 的非零梯度保证。缓解: per-topic normalization 或跳过 saturated-failure prompt 的 normalization
- **Shrinkage 对"主题内异常难题"过度惩罚**: Corollary 3.5 声称"easy topic 失败惩罚更大"，但如果某题被归为 easy topic 但实际是该 topic 的异常困难样本，会收到不恰当的大惩罚，将策略推离可能正确的方向

## 实验关键发现
- **小 group size 下优势巨大**: G=8 时 EBPO-topic 平均 65.54% vs GRPO 54.26%（+11.28%），这是最核心的发现——EBPO 在计算资源受限时最有价值
- **大 group size 下优势缩小但不消失**: G=32 时 EBPO 62.41% vs GRPO 62.91%，基本持平
- **训练动态显著改善**: 梯度范数更稳定（无消失）、KL divergence 更平滑（无 spike）、entropy 更高（更好的探索）
- **主题聚类显著提升高难度 benchmark**: AIME25 上 EBPO-topic vs EBPO-naive: Qwen3-8B +6.04%, Qwen3-14B +5.42%
- **跨模型一致有效**: Qwen3-8B (+5.67%), Qwen3-14B (+1.47%), LLaMA3-8B (+0.97%) 均有提升
- **与 DAPO、Dr.GRPO、EntropyMech 对比全面领先**: 在 Qwen3-8B 上平均 64.39% vs DAPO 52.63%, Dr.GRPO 56.67%, EntropyMech 49.18%

## 问题定位
### 解决的问题
- **GRPO 小 group size 下的 saturated failure 梯度消失**: 当 G=4/8 且所有采样均失败时，GRPO advantage 全零、梯度消失。EBPO 通过收缩 baseline 注入全局先验信号，使模型仍能从困难题中学到负梯度信号（"这个方向不行"）
- **GRPO baseline 估计的高方差**: 小 group size 下局部均值 MSE = σ²/G 很大。James-Stein 收缩严格降低 MSE：MSE_EBPO = (1-S_q) × σ²/G < σ²/G
- **异质数据分布下的在线统计维护**: 随机 shuffle prompt 导致 Welford 在线估计器在异质分布间震荡。Topic-coherent sampling 通过限制同 batch 内 prompt 来自同一主题，降低 Welford 估计方差

### 未解决的问题
- 问题: Shrinkage baseline 在 dLLM 高熵 token 分布下的有效性
  - 为什么难: EBPO 的 Gaussian 近似假设（theta_q 近似正态）在 dLLM 的 image token 分布下可能不成立（NLL>6，分布严重偏斜）。收缩因子 S_q 依赖 sigma^2 和 tau^2 的 Welford 估计，在高熵非高斯分布下估计质量可能退化
  - 潜在思路: 非参数收缩（基于分位数而非均值/方差）；分 token 类型分别维护先验（text token vs image token）；在 dLLM 上做 EBPO 的直接实验验证
- 问题: Topic-coherent sampling 如何推广到多模态场景
  - 为什么难: 文本数学推理可按代数/几何/概率聚类，但多模态场景的聚类维度更复杂——"图表+数学推理"和"自然图像+常识推理"应按视觉内容、任务类型还是推理模式聚类？
  - 潜在思路: 基于 embedding 相似度的自动聚类（而非手动定义主题）；多层次聚类（先按任务类型、再按难度）
- 问题: Welford 全局先验在非稳态 RL 训练中的滞后
  - 为什么难: RL 训练中策略持续演化（成功率随训练上升），Welford 在线估计器的 mu_glob 可能系统性滞后于当前策略的真实成功率。在 dLLM RL 中策略变化可能更剧烈（masked diffusion 探索性更强）
  - 潜在思路: 指数加权 Welford（更快遗忘旧统计量）；分阶段重置先验；自适应学习率调整 Welford 更新步长
- 问题: G=32 时优势消失的理论解释
  - 为什么难: G=32 时 EBPO 62.41% vs GRPO 62.91%（基本持平甚至略低），说明大 group size 下收缩的边际收益消失。但缺乏理论预测——在什么 G 阈值下收缩开始不如局部估计？
  - 潜在思路: 推导 MSE_EBPO vs MSE_GRPO 的交叉点作为 G 的函数；在 G=16 做中间点实验验证过渡行为

### 对问题树的推进
- 推进了 [[problem-tree#RL-2]] GRPO/PPO 如何适配多模态？: EBPO 的收缩 baseline 和主题聚类采样为多模态 GRPO 变体（UniGRPO、GSPO、Self-GRPO）提供了即插即用的改进——多模态场景的 group size 通常受限（图像生成 rollout 成本高），正是 EBPO 优势最大的区间
- 推进了 [[problem-tree#RL-3]] RL 能提升多模态推理能力吗？: EBPO 在数学推理上 +11.28%（G=8），验证了在困难推理任务上通过改进 advantage 估计可显著提升 RL 效果。多模态推理（几何、图表）同样存在高失败率的困难样本，EBPO 的机制直接适用
- 推进了 [[problem-tree#RL-3c]] 传统 VLM RL vs dLLM RL 的方法论差异: EBPO 的 shrinkage baseline 是**与架构无关的 PG 组件**——它仅改进 advantage 估计，不依赖 AR 还是 masked diffusion 的 likelihood 估计方式。这为 [RL-3c] 中"抽象出与架构无关的 PG 组件"这一潜在思路提供了首个具体实例
