## 一句话总结
提出 LFPO，一种面向 Masked Diffusion Language Model (dLLM) 的无似然度 RL 对齐方法，将 flow matching 的向量场概念映射到离散 token 空间，通过对比式速度修正（contrastive velocity rectification）直接在 logit 空间优化，绕过了传统似然度计算的高方差近似，在推理和代码任务上平均提升 10% 准确率，同时减少 20% 推理延迟。

## 核心 Insight
**dLLM RL 的根本困难不在于 reward 设计，而在于似然度估计的高方差** — 现有方法（如 UniGRPO、d1）需通过 ODE/SDE 离散化近似不可解的似然度，引入高方差和计算开销。LFPO 证明了 cross-entropy loss 对 logits 的梯度**精确等于**模型速度与目标速度之间的残差向量（Theorem 3.1），因此可以完全绕过似然度计算，直接在 logit 空间做对比式优化。这不是近似，而是精确的梯度等价。

## 方法细节

### 核心理论: 梯度对齐定理 (Theorem 3.1)
**定理**: cross-entropy loss 对 logits 的梯度精确等于模型速度与目标速度之间的残差向量:
$$\nabla_{\text{logits}} \mathcal{L}_{CE} = v_\theta(x_t, t) - v^*(x_t, t)$$
其中 $v_\theta$ 是模型预测的概率分布投影作为速度场，$v^*$ 是目标速度。

**含义**: 不需要计算概率密度/似然度——直接优化 logits 空间的速度残差即可实现策略对齐。

### Flow Matching 视角
将离散 token 提升到概率单形（probability simplex）:
- Token 表示为 one-hot 向量
- Masked 状态表示为均匀分布
- 模型预测的概率分布作为速度场投影: $v_\theta(x_t, t) := p_\theta(\cdot|x_t) - x_t$

### 对比式速度修正 (Contrastive Velocity Rectification)
两个隐式目标策略引导优化:
- **正策略 $\pi^+$**: 对高 reward 轨迹放大偏离 reference model 的方向
- **负策略 $\pi^-$**: 对低 reward 轨迹反转偏离方向

损失函数根据轨迹 reward 在正负目标之间动态插值。

**几何直觉**: 正策略"拉向"正确 token 方向（attraction），负策略"推离"低 reward 区域（repulsion），两者共同塑造速度场。消融实验证明两者缺一不可。

### 可扩展优化策略

**分层轨迹采样 (Stratified Trajectory Sampling)**: 将时间步范围划分为 K 个段，每段采样一个时间步，确保密集的时间步覆盖，同时降低方差。

**逐块梯度累积 (Block-wise Gradient Accumulation)**: 将 B×N×K 个样本划分为内存高效的块，块内并行计算、块间串行累积，维持 GPU 效率。

## Building Blocks（可复用组件）

### Block 1: 无似然度策略优化 (Likelihood-Free Policy Optimization)
- **做法**: 利用梯度对齐定理（Theorem 3.1），直接在 logit 空间通过对比式速度修正优化 dLLM 策略，完全绕过似然度估计。正/负策略分别拉向高 reward 方向和推离低 reward 区域
- **机制 (WHY it works)**:
  - Cross-entropy 梯度精确等于速度残差——不是近似，而是数学等价，避免了 ODE/SDE 离散化的近似误差
  - Flow matching 视角将优化目标从"密度估计"转换为"速度场修正"，在离散 token 空间中天然可计算
  - 对比式目标（正+负）比仅吸引或仅排斥更稳定——避免单一方向梯度导致的 policy collapse 或无效更新
  - 轨迹拉直效应: flow matching 鼓励直线去噪路径，减少所需步数
- **适用条件**:
  - 基于 masked diffusion 的语言模型（dLLM），如 LLaDA、DiffuCoder
  - 需要 verifiable reward 信号（代码 pass rate、数学准确率等）
  - 适用于推理和代码生成等需要精确对齐的任务
- **什么时候会 break**:
  - 非 masked diffusion 架构（如连续扩散、AR 模型）无法直接应用——梯度对齐定理依赖离散 token 空间的特性
  - 当 reward 信号极其稀疏或噪声极大时，正/负策略的对比信号可能退化
  - 多模态场景（image token + text token）的高熵分布下，logit 空间的速度修正精度是否仍然足够——需要验证
- **可组合方向**:
  - 可替换 KB 中所有 dLLM RL 方法的核心优化模块（UniGRPO、d1、DiffuGRPO、Self-GRPO 的 likelihood estimation 部分）
  - 与 [[2026-EBPO]] 的 shrinkage baseline 正交组合——LFPO 改进梯度估计精度，EBPO 改进 advantage baseline 估计
  - 与 [[2026-LaViDa-R1]] 的 answer-forcing 结合——LFPO 提供精确梯度，answer-forcing 解决困难问题的训练信号消失

### Block 2: 分层轨迹采样 (Stratified Trajectory Sampling)
- **做法**: 将扩散时间步范围 [0, 1] 划分为 K 个等间距段，每段内随机采样一个时间步，确保每次更新都均匀覆盖整个去噪过程
- **机制 (WHY it works)**:
  - 消除了 i.i.d. 随机采样可能造成的时间步覆盖不均——某些关键去噪阶段（如早期全局结构决策）可能被漏掉
  - 减少了梯度估计方差——均匀覆盖使每次梯度更新包含去噪过程所有阶段的信息
  - 与 UniGRPO 的"结构化随机 mask ratio"思路相似但更系统化
- **适用条件**: 任何基于多步去噪的 dLLM RL 训练
- **什么时候会 break**: K 过大时每段只有一个时间步，采样粒度过细反而引入开销；去噪步数很少时（<10 步）分层意义不大
- **可组合方向**: 可直接替换 UniGRPO 的随机 mask ratio 采样策略；与 LaViDa-R1 的 complementary masking 正交（一个优化时间步覆盖，一个优化 token 覆盖）

### Block 3: 逐块梯度累积 (Block-wise Gradient Accumulation)
- **做法**: 将 B×N×K 个 RL 样本划分为内存高效的块，块内并行前向传播和梯度计算，块间串行累积梯度，最后统一更新参数
- **机制 (WHY it works)**: 解决了 dLLM RL 的显存瓶颈——每个 prompt 的 N 个 rollout × K 个时间步 = NK 次前向传播的显存需求。通过分块处理，显存开销从 O(BNK) 降到 O(block_size)
- **适用条件**: 大 batch RL 训练（B 大）或多 rollout 采样（N 大）场景
- **什么时候会 break**: 块太小时串行开销增加，效率下降
- **可组合方向**: 与 DiMOO 的 Step Trajectory Following 思路相似但作用维度不同（STF 选择时间步子集，block-wise 分块计算全部样本）；两者可结合实现双维度显存优化

## Anti-patterns / 已知失败模式
- **Theorem 3.1 在连续时间极限外的离散化误差**: 梯度对齐定理依赖连续时间 CTMC 的速度定义，实际训练使用有限时间步近似。在少步推理（8-32步）场景下，离散化误差可能系统性偏大
- **高熵 token 分布下速度信号可能被噪声淹没**: LaViDa-R1 已报告 image token NLL>6。虽然 LFPO 绕过了 likelihood 估计，但 logit 空间的速度残差在模型对 image token 预测置信度低（高 entropy）时方向高度不确定，正/负策略的对比信号微弱
- **Token 级 credit assignment 被牺牲**: likelihood-based 方法（UniGRPO、complementary masking）可通过 token 级 log prob 做细粒度 credit assignment。LFPO 在时间步级别操作速度残差，token 级信号被平均化。在关键推理步骤仅涉及 2-3 个 token 的场景下可能不如 likelihood-based 方法
- **EMA reference model 在非稳态训练中的滞后**: 与 EBPO 的 Welford 在线估计器滞后问题同源——EMA decay rate 固定时，训练早期策略快速演化导致 reference model 系统性滞后，偏离方向过时。不同模态可能演化速度不同（文本推理早期快速提升 vs 图像生成缓慢），单一 EMA rate 是次优的
- **消融显示正/负策略的信息冗余度较高**: 仅正策略 58.9、仅负策略 59.0，组合 60.8——两者单独效果几乎相等，组合提升仅 +1.8，远非 additive，说明共享大量信息。对比式目标的"必要性"可能不如论文声称的那么强
- **缺乏显式 KL 约束**: LFPO 既不使用 KL penalty（UniGRPO）也不使用 SFT 正则化（LaViDa-R1）。EMA + 速度对比提供隐式约束，但无法显式量化策略偏离程度。在需要保守策略更新的部署场景下控制精度不足
- **Saturated failure 场景（全零 reward）**: LFPO 的负策略需要"反转偏离方向"——当所有采样都得零分时，正/负划分失效。EBPO 的 shrinkage baseline 在此场景下严格更优（通过全局先验注入信号），LFPO 未解决此问题

## 实验关键发现
- **代码生成 (DiffuCoder 基座)**:
  - HumanEval: 75.6（+3.6% absolute 提升）
  - 跨基准平均: 60.8，超越 AGRPO (60.6)、Coupled-GRPO、SPG、UniGRPO
- **推理任务 (LLaDA 8B 基座)**:
  - GSM8K: 79.6（+9.9% over base）
  - MATH: 37.6（+7.0% over base）
  - 跨基准平均: 67.1 vs AGRPO 66.6
- **推理效率显著提升**:
  - 代码任务: 平均减少 41.8 步
  - 推理任务: 平均减少 159.0 步
  - 对比 AGRPO 在 MATH 上增加 +73.6 步——LFPO 减步，AGRPO 增步
- **训练收敛速度**:
  - HumanEval/MATH 上达到基线性能快 8.0×
  - Hellaswag 上快 4.4×
- **消融实验**:
  - 仅正策略 (attraction): 代码 58.9, 推理 64.6
  - 仅负策略 (repulsion): 代码 59.0, 推理 64.5
  - 正+负 (contrastive): 代码 60.8, 推理 67.1
  - **正负策略缺一不可**——单独使用效果相近但均明显弱于组合

- `alternative_to` → [[2026-LaViDa-R1]]: 解决不同问题的 dLLM RL 优化——LFPO 梯度精确性 vs LaViDa-R1 训练信号消失；LFPO 的 likelihood-free gradient 可替代 LaViDa-R1 的 complementary masking 估计

- `combines_with` → [[2026-LaViDa-R1]]: LFPO 精确梯度 + LaViDa-R1 answer-forcing 可形成"精确梯度 + 智能采样"的双层优化
- `combines_with` → [[2025-XDLM]]: 同为从 flow matching 视角重新理解离散扩散——LFPO 后训练减步（轨迹拉直）+ XDLM 预训练减步（混合噪声核），机制正交可叠加

- `enables` → 任意 masked diffusion LM 的 RL 优化: 无似然度框架仅需 cross-entropy + softmax，可直接适配 LLaDA 系列、Dream-DLM 等任何离散扩散模型

## 问题定位
### 解决的问题
- **dLLM RL 中似然度估计的高方差问题**: 现有方法（UniGRPO、d1、DiffuGRPO）需通过 ODE/SDE 离散化或蒙特卡洛采样近似不可解的似然度，引入高方差和计算开销。LFPO 通过 Theorem 3.1 证明 cross-entropy 梯度精确等于速度残差，完全绕过似然度计算，是精确解而非近似
- **dLLM RL 训练收敛速度慢**: 高方差梯度估计导致收敛缓慢。LFPO 在 HumanEval/MATH 上达到基线性能快 8.0×，在 Hellaswag 上快 4.4×
- **dLLM 推理步数过多**: 现有 dLLM RL 方法（如 AGRPO）训练后反而增加推理步数。LFPO 的 flow matching 轨迹拉直效应减少推理步数（代码 -41.8 步，推理 -159.0 步），同时提升质量
- **开辟 dLLM RL 的第四范式**: 在 UniGRPO（结构化随机近似）、complementary masking（互补掩码降方差）、EBPO（shrinkage baseline）之后，LFPO 提出了基于速度场修正的无似然度范式，从根本上改变了优化目标的表述方式

### 未解决的问题
- 问题: 多模态场景（image token + text token）下速度修正的有效性
  - 为什么难: Theorem 3.1 在离散 text token 上推导，但 image token 的高熵分布（NLL>6，LaViDa-R1 报告）使 logit 空间的速度残差方向高度不确定。正/负策略的对比信号在高熵下可能被噪声淹没
  - 潜在思路: 分 token 类型应用不同强度的速度修正（text token 精确修正、image token 渐进修正）；与 XDLM 的混合噪声核结合降低 image token 的初始熵
- 问题: Token 级 credit assignment 的缺失
  - 为什么难: LFPO 在时间步级别操作速度残差，token 级信号被平均化。当关键推理步骤仅涉及 2-3 个 token 时（如数学推理的关键等号），时间步级优化无法精确归因
  - 潜在思路: 引入 token-level attention weight 对速度残差做加权；与 complementary masking 的 token 级 log prob 信号结合
- 问题: 轨迹拉直效应的理论解释
  - 为什么难: LFPO 实验观察到推理步数显著减少（代码 -41.8 步，推理 -159 步），论文归因于 flow matching 的轨迹拉直效应，但缺乏严格的理论分析——为什么对比式速度修正会导致更直的去噪路径？
  - 潜在思路: 分析对比式优化目标的隐式正则化效应；与 XDLM 的 k=0.1 混合噪声核（也减少步数）做机制对比
- 问题: 缺乏显式 KL 约束的安全性
  - 为什么难: LFPO 既不使用 KL penalty 也不使用 SFT 正则化，仅依赖 EMA + 速度对比提供隐式约束。在需要保守策略更新的部署场景（如安全对齐）中，无法显式量化策略偏离程度
  - 潜在思路: 将 LFPO 的速度残差范数作为 KL 的代理指标；设计基于速度场距离的显式约束
- 问题: Saturated failure 场景的处理
  - 为什么难: 当所有采样都得零分时，正/负策略划分失效（无正样本）。EBPO 的 shrinkage baseline 在此场景下严格更优——通过全局先验注入负梯度信号
  - 潜在思路: 直接与 EBPO shrinkage baseline 组合——LFPO 负责精确梯度，EBPO 负责全零场景的信号注入

### 对问题树的推进
- 推进了 [[problem-tree#RL-2]] GRPO/PPO 如何适配多模态？: LFPO 提供了第四种 dLLM RL 范式（无似然度优化），与 UniGRPO（近似似然度）、complementary masking（降方差似然度）、EBPO（shrinkage baseline）正交。更重要的是，LFPO 的无似然度特性天然规避了 KL 估计在高熵 image token 下的方差问题（[[2026-LaViDa-R1]] 报告的 NLL>6 问题），为 [RL-2] 的多模态扩展提供了新的可能性
- 推进了 [[problem-tree#RL-3c]] 传统 VLM RL vs dLLM RL 的方法论差异: LFPO 是第二个**仅适用于 dLLM 而非 AR 模型**的 RL 组件（第一个是 LaViDa-R1 的 answer-forcing）。Theorem 3.1 依赖离散 token 空间 + cross-entropy loss 的特性，无法直接迁移到 AR 模型。这进一步证实了 [RL-3c] 的判断：dLLM RL 正在发展出独立于 AR RL 的方法论体系
- 推进了 [[problem-tree#Diff-1b]] 连续扩散 vs 离散扩散: LFPO 将 flow matching（连续扩散理论）的速度场概念成功映射到离散 token 空间，提供了连续-离散之间的桥接证据。与 [[2025-XDLM]] 的 Gaussian kernel 混合类似，LFPO 表明连续扩散理论可以为离散扩散提供有用的分析工具
- 推进了 [[problem-tree#Diff-1c]] 推理加速: LFPO 的轨迹拉直效应提供了一种**新的推理加速维度**——通过 RL 后训练减少推理步数（代码 -41.8 步，推理 -159 步）。这与 XDLM 的预训练层面减步（混合噪声核）、DiMOO 的推理层面减步（ML-Cache）形成三层加速体系
- 新增问题:
  - **[RL-2g] 🔴 Velocity-based vs Likelihood-based dLLM RL 的系统对比**: LFPO 的速度场修正和 UniGRPO/complementary masking 的似然度估计代表两条根本不同的优化路径。需要在相同基座、相同任务、相同 reward 下做严格对比——精度 vs 方差 vs 计算成本 vs 推理步数的全面 Pareto 分析
  - **[RL-2h] 🔴 RL 后训练的轨迹拉直机制**: LFPO 观察到 RL 训练后推理步数减少，但缺乏理论解释。这是 flow matching 的固有性质还是对比式优化的特异效果？能否设计专门利用此效应的训练目标，进一步放大减步效果？

### [Critic] 关键隐含假设
1. **连续时间极限假设**: Theorem 3.1 在连续时间 CTMC 框架下推导，实际训练使用有限时间步。少步推理（8-32 步）场景下离散化误差可能系统性偏大
2. **速度场平滑性假设**: 对比式速度修正假设速度场在 logit 空间中足够平滑，使得正/负方向的线性插值有意义。在高熵 token 分布下（image token），速度场可能高度非平滑
3. **正/负策略对称性假设**: 消融显示正策略 58.9、负策略 59.0，效果几乎对称。但组合提升仅 +1.8（60.8），远非 additive，说明两者共享大量信息。对比式目标的"必要性"可能不如论文声称的那么强
4. **EMA 稳定性假设**: 固定 decay rate 的 EMA 假设策略演化速度恒定。训练早期策略快速变化时，EMA reference 系统性滞后，与 EBPO 的 Welford 滞后问题同源

### [Ideator] 三条技术发展线交汇定位
1. **dLLM RL 路线**: d1 → UniGRPO (MMaDA) → DiffuGRPO → complementary masking (LaViDa-R1) → **LFPO** (无似然度范式)
2. **Flow Matching 应用线**: 连续扩散 flow matching → XDLM (混合噪声核) → Beyond-LM (连续视觉 FM) → **LFPO** (离散 token FM 后训练)
3. **RL 方差降低线**: GRPO → DAPO/Dr.GRPO → EBPO (shrinkage baseline) → **LFPO** (精确梯度绕过方差来源)

LFPO 的核心学术价值: **将优化目标从"估计密度"转换为"修正速度场"**——不是用更好的近似解决方差问题，而是从根本上改变问题的表述方式，使方差问题不再存在。

### [Ideator] 潜在研究方向
1. **LFPO + EBPO 双层方差降低**: LFPO 精确梯度（消除似然度估计方差）+ EBPO shrinkage baseline（降低 advantage 估计方差），作用于 RL 管线不同环节，理论上可叠加。实验验证: 在 LLaDA 8B 上用 LFPO 优化器 + EBPO advantage baseline，对比各自单独效果
2. **LFPO + XDLM 双层推理加速**: XDLM 预训练减步（k=0.1 混合噪声核）+ LFPO 后训练减步（轨迹拉直），机制正交——前者改变噪声分布使去噪更容易，后者 RL 训练使去噪路径更直。潜在实现: 用 XDLM 预训练的 dLLM 做 LFPO 后训练
3. **LFPO 应用于 T2I RL 对齐**: LFPO 的无似然度特性天然规避 KL 估计在高熵 image token 下的方差问题。配合 Beyond-LM 的 flow matching 视觉目标和 LaViDa-R1 的 EditScore reward，可能是首个稳定的统一模型 T2I RL 对齐方案

### [Ideator] 对 patterns 的影响
- **P-Diff-08 (推理加速)**: 应从六维扩展到七维——增加"RL 后训练轨迹拉直"维度（LFPO 代码 -41.8 步，推理 -159 步）
- **候选新 pattern**: "dLLM RL 的四种范式"——似然度近似(UniGRPO) / 似然度降方差(complementary masking) / advantage 降方差(EBPO) / 无似然度(LFPO)，各有适用场景
