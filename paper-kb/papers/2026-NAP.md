## 一句话总结
诊断了扩散语言模型（DLMs）收敛为 AR-like 左到右解码的根因——训练数据的隐式顺序性（sequential dependence），提出 NAP（Non-Autoregressive Parallel DLMs）通过"并行推理数据重构 + 并行强制解码策略"的数据-解码协同设计，在低步数（高并行度）下显著超越标准 long-CoT 训练（GSM8K 256步: +14.4%）。

## 核心 Insight
DLMs 虽然架构上支持并行生成，但因为盲目复用为 AR 模型设计的训练数据（具有强 left-to-right 顺序依赖），模型学会了"优先从左到右解码"的 AR-like 策略。解决方案不在解码端（已有方法如 confidence-based 解码仍收敛为 AR），而在**数据-解码协同设计**——训练数据需要从根本上消除 privileged ordering（用多条独立推理轨迹替代单一 CoT 链），配合强制并行解码策略分配 unmask 预算到多个独立推理流。

## 方法细节

### 问题诊断：DLM 的 AR 收敛

#### 度量工具
1. **Global ARness@k**: 衡量解码过程中 unmask 顺序与 left-to-right 的一致度。值域 [0,1]，1.0 表示严格 AR 解码
   - $\mathbb{I}_{global}(c,k) = 1 \text{ if } p_k \in \text{smallest-k}(M_{k-1})$

2. **Sequential Dependence (SeqDep)**: 衡量当前 text segment 对前文的依赖程度（vs 仅依赖初始 prompt）
   - $\text{SeqDep}(x, \mathbf{s}) = \frac{1}{N-1}\sum_{n=2}^{N}[\log p_{AR}(s_n|x, s_{<n}) - \log p_{AR}(s_n|x)]$
   - 接近 0 表示 segments 条件独立（可并行），高值表示强链式依赖

#### 四项实证发现
1. **训练数据高度顺序化**: FineWeb 预训练数据和 Open-R1-Math long-CoT 数据均显示强 SeqDep
2. **DLM 解码收敛为 AR**: LLaDA 和 Dream 即使使用 confidence-based Arbitrary Order 解码，unmask 顺序仍呈严格线性对角线模式
3. **Long-CoT 训练加剧 ARness**: long-CoT 后训练进一步增加 ARness——step-by-step 推理链天然具有 privileged ordering
4. **Fast-DLM 方法加剧顺序性**: 现有加速方法（block-wise 解码、early-commit 等）通过"先稳定前缀再生成后续"获得速度提升，本质上是放大而非消除 AR-like 行为

### NAP 方法设计

#### 数据重构：并行推理轨迹
1. 给定 query x，用高温度（τ=1.0）的教师模型生成 P 条独立推理轨迹 {r⁽¹⁾, ..., r⁽ᴾ⁾}
2. 将多条轨迹 + 汇总块打包为单一训练实例:
   - Y = [B₁, R⁽¹⁾, B₂, R⁽²⁾, ..., Bₘ, R⁽ᵐ⁾, Bₛ, S]
   - Bⱼ 为固定文本头（如 `<think #j>`），R⁽ʲ⁾ 为第 j 条推理内容，S 为汇总+答案块
3. 多条轨迹的条件独立性从根本上消除 privileged ordering——任何轨迹都不比其他轨迹更"先"

#### 并行强制解码（Parallel-Forced Decoding）
1. **Canvas 结构**: 固定 header clamped，reasoning slots 初始化为 [MASK]，m 条推理块 + 1 个汇总块
2. **宏观并行**: 每一步的 unmask 预算在所有 m 个推理块之间均匀分配，阻止"先稳定上游路径、再启动下游路径"
3. **微观置信度**: 每个推理块内部按 confidence-based 策略 unmask——高置信度 token 优先 commit

### 训练配置
- 基座: LLaDA-8B-Instruct / Dream-7B-Instruct
- 标准 masked diffusion 目标训练 3 epochs
- AdamW, lr=2e-6, batch size 256
- 硬件: 8× NVIDIA A800 GPUs
- 数据: ~100K 样本（后训练 proof-of-concept）

## Building Blocks（可复用组件）

### Block 1: ARness / SeqDep 诊断度量
- **做法**: 提出 Global ARness@k 和 Sequential Dependence (SeqDep) 两个度量，分别量化解码过程的 AR 程度和训练数据的顺序依赖程度
- **机制 (WHY it works)**: ARness 直接测量 unmask 顺序与 left-to-right 的偏差——若模型真正学会了并行解码，unmask 顺序应无方向偏好。SeqDep 使用外部 AR scorer 测量"前文对当前 segment 的信息增益"——若 segments 条件独立则 SeqDep≈0，数据天然支持并行生成
- **适用条件**: 任何 masked diffusion 模型的解码行为分析；评估训练数据是否适合并行解码
- **什么时候会 break**: (1) ARness 度量假设 left-to-right 是唯一的"顺序性"模式——如果模型学会了其他顺序模式（如 right-to-left 或 center-out），ARness 为 0 但仍非并行；(2) SeqDep 依赖外部 AR scorer 质量——弱 scorer 可能低估顺序依赖
- **可组合方向**: 作为 dLLM 训练/数据质量的通用评估工具；扩展为可微的训练正则化项（最小化 ARness）

### Block 2: 并行推理轨迹数据重构
- **做法**: 用高温教师模型为每个问题生成 P 条独立推理路径，打包为单一训练实例 Y = [B₁, R⁽¹⁾, ..., Bₘ, R⁽ᵐ⁾, Bₛ, S]。汇总块 S 聚合多条路径的 insights 得出最终答案
- **机制 (WHY it works)**:
  - **消除 privileged ordering**: 多条独立轨迹之间没有因果依赖（R⁽ⁱ⁾ 和 R⁽ʲ⁾ 条件独立于 prompt），模型无法从数据中学到"先生成哪条路径"的偏好
  - **内部 ensemble**: 汇总块强迫模型学会从多条可能含噪声的路径中提取有效信息，过滤错误推理，形成隐式多数投票机制
  - **SeqDep 平坦化**: 实验证实并行数据的 SeqDep 保持平坦（≈12），不随序列长度增长，与标准 long-CoT 的单调递增形成对比
- **适用条件**: 有可验证答案的推理任务（数学、逻辑、代码）；教师模型能生成多条正确的独立推理路径
- **什么时候会 break**: (1) 某些问题只有唯一推理路径（如线性证明），强行生成"独立"路径会引入低质量的伪多样性；(2) 教师模型多样性不足时多条路径实质重复；(3) 汇总块成为新的顺序瓶颈——所有路径仍需按顺序排列在汇总块之前
- **可组合方向**: 与 [[2026-LaViDa-R1]] 的 answer-forcing 结合——NAP 提供并行推理数据，answer-forcing 提供困难问题的训练信号恢复；与 XDLM k=0.1 结合——混合噪声核训练 + 并行数据，双层解决 AR 收敛

### Block 3: Parallel-Forced Decoding（并行强制解码）
- **做法**: 将输出结构化为 m 个独立推理块 + 1 个汇总块。宏观层面在每步将 unmask 预算均匀分配到所有推理块；微观层面每个块内按 confidence 排序 unmask
- **机制 (WHY it works)**:
  - **打断顺序临界路径**: 均匀分配 unmask 预算阻止模型建立"上游→下游"的依赖链——所有推理流同时进展
  - **利用条件独立性**: 推理块之间条件独立（given prompt），并行 unmask 不会因缺乏跨块上下文而损失信息
  - **兼容 micro-level 灵活性**: 块内 confidence-based 解码保留有效的 token 选择策略
- **适用条件**: 训练数据具有并行结构（如 NAP 数据重构后）；输出可分解为多个独立子任务
- **什么时候会 break**: (1) **必须与 NAP 数据协同使用**——对未经并行数据训练的标准模型施加 parallel-forced decoding 会导致性能下降（Table 4: Dream-7B base + PF = 31.0% vs AO = 35.0% at 256 steps），因为标准模型的隐式顺序偏好被强行打断；(2) 输出结构无法自然分解为独立子块的任务（如叙述性文本生成）
- **可组合方向**: 扩展到多模态场景——图像和文本作为不同的并行流（类比 [[2025-MMaDA-Parallel]]）；与 [[2025-XDLM]] 少步推理结合（减少步数 × 每步更多 token）

## Anti-patterns / 已知失败模式
- **Confidence-based AO 解码在标准数据训练的 DLM 上仍收敛为 AR**: Figure 1(a,b) 显示 LLaDA 和 Dream 的 AO 解码呈严格线性对角线，说明解码策略本身无法解决 AR 收敛——根源在数据
- **Long-CoT 后训练加剧 ARness**: 标准 long-CoT SFT 进一步强化 left-to-right 偏好，与 DLM 的并行潜力矛盾
- **Fast-DLM 方法本质是 AR 加速而非并行加速**: Block-wise 解码（如 MMaDA）、early-commit（如 Prophet）通过"先稳定前缀再生成后续"获得加速，实质在 amplify 而非 remove AR-like behavior
- **Parallel-Forced Decoding 单独使用（无 NAP 数据）导致性能下降**: Table 4 显示标准基础模型 + PF 解码 = 性能下降，证明数据-解码必须协同设计
- **[Critic] 仅 post-training 规模验证（~100K 样本），未验证预训练层面的并行化效果**: 论文自身承认这是 proof-of-concept，完全消除 AR 瓶颈可能需要在预训练阶段使用非顺序结构化数据
- **[Critic] 教师模型准确率阈值风险**: 并行轨迹数据依赖教师模型（高温 τ=1.0）生成多条正确路径。若教师准确率不足（如困难 MATH 题），多条"独立"路径可能均为错误解法，汇总块学到的是噪声聚合而非有效 ensemble
- **[Critic] 汇总块的被动多数投票**: 汇总块 S 被训练为从多条路径提取信息，但其机制更接近 pattern-matching "最常见结论" 而非真正的 cross-validation——当多条路径犯同一系统性错误时，汇总块无法纠正
- **[Critic] 序列长度 O(m) 膨胀**: m 条并行轨迹打包到单一序列，总长度 = m × 单轨迹长度 + 汇总块。m=3 时序列长度约为标准 CoT 的 3.5×，训练 FLOPs 按 O(L²) 增长（full attention），实际训练成本增长 ~12×。论文未充分讨论此开销与性能提升的性价比
- **[Critic] 缺少 self-consistency 基线对比**: m 条独立轨迹 + 汇总块的性能提升可能部分来自 "多次采样+多数投票" 的 self-consistency 效应，而非真正的并行解码改善。应对比: m×独立采样 + 外部投票 vs NAP 数据训练 + PF 解码，以分离 ensemble 效应和并行化效应
- **[Critic] 与 RL 后训练的兼容性未验证**: NAP 数据格式（多轨迹+汇总块）是否与 UniGRPO/StableDRL 等 dLLM RL 兼容？RL 的 advantage 估计需要跨多个完整 response 比较，NAP 的并行结构增加了 credit assignment 复杂度

## 实验关键发现
- **NAP 在低步数下优势最大**: Dream-7B GSM8K 256步(4tok/step): NAP 60.9% vs Long-CoT 46.5% (+14.4%)；1024步(1tok/step): NAP 83.6% vs Long-CoT 78.0% (+5.6%)
- **并行度越高，NAP 优势越大**: 从 1024 步 → 256 步，NAP vs Long-CoT 差距从 +5.6% 扩大到 +14.4%（GSM8K）
- **数据-解码协同是必要的**: Table 4 显示单独使用 NAP 数据（AO 解码）或单独使用 PF 解码（标准数据）都不如 NAP 完整方案
- **增加并行路径数持续提升性能**: m=1→3 时 Dream-7B GSM8K 从 75.4% → 83.6%（internal ensemble 效应）
- **并行数据的 SeqDep 平坦**: mean≈12 保持稳定不随序列长度增长，与标准 long-CoT 的递增依赖形成对比
- **NAP 解码动态呈明显并行带状模式**: Figure 1(d) 显示 NAP 的 unmask 顺序呈多条并行带，确认多条推理轨迹同时生成

## 问题定位
### 解决的问题
- **首次系统性诊断 DLM 的 AR 解码收敛现象并提出定量度量（ARness, SeqDep）**: 揭示了 DLM "名义上并行但实际上 AR" 的根本原因——训练数据的隐式顺序结构，而非架构或解码策略的限制
- **提出数据-解码协同的解决框架**: 证明单独改变解码策略无效（标准数据 + PF = 性能下降），单独改变数据也不够（NAP 数据 + AO 虽大幅提升但仍有提升空间），二者必须协同
- **在高并行度 regime 下显著提升推理性能**: 256 步（4 tok/step）下 GSM8K +14.4%，使 DLM 在少步推理下更接近其理论并行潜力

### 未解决的问题
- 问题: 预训练阶段的并行化
  - 为什么难: 当前仅在后训练（~100K 样本）验证，标准预训练语料（FineWeb 等）具有极强 SeqDep，完全重构预训练数据到并行结构代价巨大且并非所有文本都有自然的并行分解
  - 潜在思路: 混合预训练（部分并行数据 + 部分标准数据）；curriculum（预训练中后期引入并行数据）；对特定任务型语料做并行重构（如多步数学证明→并行证明路径）
- 问题: 汇总块的顺序瓶颈
  - 为什么难: 即使推理轨迹并行，汇总块 S 仍位于所有轨迹之后，形成新的顺序瓶颈。模型需要先"完成所有推理"再"汇总"——这本身是顺序的
  - 潜在思路: streaming 汇总（边推理边汇总）；去除显式汇总块，训练模型在最后一步直接从多条轨迹输出答案
- 问题: 多模态场景的并行化
  - 为什么难: 当前仅验证纯文本推理。多模态场景（如图文推理→图像生成）的 sequential 依赖更复杂——文本推理和图像生成可能有因果依赖
  - 潜在思路: 与 [[2025-MMaDA-Parallel]] 的 interleaved 架构结合——文本推理流和图像生成流作为两个并行 canvas 块
- 问题: 对非推理型任务的适用性
  - 为什么难: 叙述性文本（故事、描述）天然是顺序的，无法自然分解为独立推理路径。NAP 的内部 ensemble 机制对非推理任务是否有效存疑
  - 潜在思路: 段落级并行（不同段落独立生成再连接）；层次化并行（outline 并行生成→每段内容并行生成）

### 对问题树的推进
- 推进了 [[problem-tree#Diff-1b 连续扩散 vs 离散扩散]]: 揭示 masked diffusion 的一个被严重低估的问题——AR 解码收敛，并提出首个系统性解决方案。为 Diff-1b 新增子问题维度
- 推进了 [[problem-tree#Diff-1c 采样效率]]: NAP 在低步数 regime 下显著提升质量，使 DLM 在少步推理下更接近理论潜力。NAP 是 P-Diff-08 推理加速的**第十个正交维度**——"并行数据+并行解码"（前九个维度: Prefix-DLM, ML-Cache, mask 截断, MARS-Cache, XDLM 少步, block-wise, early-commit, LFPO 轨迹拉直, Semi-AR）
- 推进了 [[problem-tree#Diff-1k MDLM 训练饱和的根因]] → 部分解决: NAP 从数据端提供新解释——MDLM 饱和可能不仅是"训练目标过简单"或"训练-推理分布不匹配"，更是"数据的隐式顺序结构限制了模型利用并行能力的学习空间"。与 XDLM（噪声核解释）和 ReDiff（训练-推理 gap 解释）形成三种互补假设
- 新增问题: [Diff-1l] 🔴 DLM 的 AR 解码收敛是否可在预训练阶段解决？并行结构化预训练数据的可行性和成本？

## 个人深度评注

### [Critic] 增量贡献评估
| Block | 新颖性 | 理由 |
|-------|--------|------|
| Block 1: ARness / SeqDep 诊断度量 | 中高 | 首次为 dLLM 的 AR 收敛现象提供定量度量框架。ARness 度量本身是直觉的指标化（"unmask 顺序是否 L→R"），但 SeqDep 引入外部 AR scorer 量化数据的并行潜力更有新意。主要价值: 为后续所有 dLLM 并行化工作提供标准化评估工具 |
| Block 2: 并行推理轨迹数据重构 | 中 | 核心思想（多条独立路径消除 privileged ordering）直觉上自然，类似 self-consistency 的训练时版本。**关键质疑**: 性能提升中有多少来自 "m× ensemble 效应"（与 self-consistency 等效）vs "真正的并行解码能力提升"？论文缺少关键消融: m×独立采样+多数投票 vs NAP。序列长度 O(m) 膨胀带来的 ~12× 训练 FLOPs 增长也削弱了性价比 |
| Block 3: Parallel-Forced Decoding | 低中 | 给定 NAP 数据结构后的"自然选择"——均匀分配 unmask 预算到独立推理块。缺乏理论保证（为何均匀是最优？）。核心贡献更在于"必须与 NAP 数据协同"的实验发现（Table 4），而非解码策略本身 |

**总体判断**: NAP 最大的贡献是**诊断框架**（Block 1）——首次系统性揭示 dLLM AR 收敛现象并提供量化工具。解决方案（Block 2+3）的新颖性相对有限，更多是"如果数据有并行结构，解码也应该并行"的直觉实现。核心悬疑是 ensemble vs parallelization 效应的分离。

### [Critic] 关键隐含假设
1. **轨迹条件独立性**: 假设教师模型（高温 τ=1.0）生成的多条推理路径真正条件独立于 prompt。实际上同一教师模型的输出在特征空间中往往相关——不同温度采样的路径可能共享推理起点或中间步骤，"条件独立"只是近似
2. **教师准确率充分**: 并行数据质量完全依赖教师模型。在困难问题（如 MATH Level 5）上，高温采样可能多数路径错误，汇总块学到的是噪声聚合
3. **汇总块不构成瓶颈**: 虽然推理轨迹并行，汇总块 S 仍位于所有轨迹之后且是顺序生成的。当推理链很长或答案依赖复杂的跨路径信息融合时，汇总块成为新的顺序瓶颈
4. **数学推理任务代表通用场景**: 所有实验仅在 GSM8K/MATH 上验证。代码生成（函数间有调用依赖）、逻辑推理（前提链式推导）、文本续写等任务的"可并行分解性"可能远低于数学
5. **后训练效果可外推到预训练**: 论文建议在预训练阶段使用并行数据，但 ~100K 后训练 proof-of-concept 不能直接外推——预训练语料的多样性和规模与后训练根本不同
6. **ARness↔性能因果关系**: 论文证明了相关性（高 ARness 的模型在低步数下性能差，低 ARness 性能好），但未严格证明因果关系——可能存在混淆变量（如 NAP 数据本身的多样性/ensemble 效应独立于 ARness 降低贡献了性能提升）

### [Critic] 机制层深度分析

**NAP 与 P-Diff-04（训练-推理分布不匹配）的理论连接**:
NAP 的数据重构从另一个角度缓解了 P-Diff-04 描述的训练-推理分布不匹配问题。标准 dLLM 训练时，模型在 clean context + [MASK] 条件下预测，推理时面对"部分正确+部分错误"的中间状态。NAP 的并行推理块在训练时就暴露模型于"多个不完整推理流同时存在"的中间状态——这与推理时的并行去噪状态更接近。

这形成了解决 P-Diff-04 的**四层方案谱系**:
1. **噪声核层** (XDLM): 训练时注入 uniform 噪声使模型 robust
2. **训练策略层** (ReDiff): 显式注入合成错误训练纠错能力
3. **数据层** (NAP): 并行数据结构使训练分布更接近推理分布
4. **架构层** (MMaDA-Parallel): 并行架构避免 sequential 错误传播

**"数据决定论" vs "架构决定论" 的张力**:
NAP 的核心主张是"AR 收敛的根源在数据而非架构/解码"——这是一个强假设。与 XDLM 的"根源在噪声核设计"和 ReDiff 的"根源在训练-推理 gap"形成三种竞争假说。更可能的现实是三者共同作用: 数据顺序性 → 模型学会 AR 偏好 → 少步推理时残余噪声放大偏好 → 性能退化。NAP 在数据层面解决最表层症状，XDLM/ReDiff 解决更深层机制。

### [Ideator] 潜在研究方向

1. **ARness 正则化训练** (高价值、中风险): 将 ARness 度量转化为可微的训练正则化项 $\mathcal{L}_{reg} = -\lambda \cdot H(\text{unmask order})$，在标准 masked diffusion 训练中鼓励均匀解码顺序。优势: 无需重构数据，可在任何 dLLM 预训练/后训练中 drop-in。风险: unmask order 的采样过程可能难以反向传播梯度
2. **NAP + XDLM 双层方案** (高价值、低风险): 在 XDLM k=0.1 混合噪声核预训练的基础上，用 NAP 并行数据做后训练。两者从不同层面解决 AR 收敛——XDLM 从噪声层面使模型 robust，NAP 从数据层面消除顺序依赖。在 LLaDA-8B continual pretraining (XDLM k=0.1) → NAP 后训练管线上验证
3. **NAP + RL (StableDRL/EBPO)** (中高价值、中风险): 将 NAP 并行推理轨迹 SFT 作为 RL 冷启动，然后用 StableDRL（稳定梯度更新）或 EBPO（shrinkage baseline）做 RL 后训练。关键挑战: NAP 的多轨迹+汇总块结构与 GRPO reward 的兼容性——可能需要将每条轨迹视为独立 response 分别评分，汇总块答案作为最终 reward
4. **预训练阶段的并行化** (高价值、高风险): 对 FineWeb 等预训练语料做并行结构重构——对于 list-style 内容（对比、枚举）自动检测并重排为并行格式；对 narrative 内容保持原始顺序。混合比例 curriculum: 前期 100% 标准数据，后期渐增并行数据到 20%。风险: 规模和成本；并行重构质量
5. **多模态并行化** (中价值、中风险): 将 NAP 的并行推理流扩展到多模态——文本推理流和图像生成流作为两个并行 canvas 块（类比 MMaDA-Parallel 但从数据端解决）。在 LaViDa-O 或 DiMOO 上验证

### [User] 个人评注

**NAP 的根本局限性**: NAP 从数据层面解决并行生成，类似 Agent 系统中启动多个 sub-Agent 并行执行——这种方式虽然提供了一个潜在思路，但有非常大的局限性：

1. **对数据要求极高**: 需要教师模型能生成多条高质量独立推理路径，困难问题上难以满足
2. **序列长度膨胀**: m 条轨迹打包导致 O(m) 序列长度增长，训练成本 ~12× 增长
3. **治标不治本**: 数据层面的并行化绕过了 dLLM 架构本身的并行生成能力问题

**更深层的思考方向**: 如果要真正解决并行生成，可能需要从 **MASK 机制本身** 重新思考。当前 masked diffusion 的 [MASK] token 方式的合理性值得质疑——也许存在某些**隐变量**能使解码时一次生成多个 token，而非逐步 unmask。这个想法尚不成熟，但可能是比数据重构更根本的解决路径。

NAP 的价值更多在于**诊断框架**（ARness/SeqDep ���量）而非解决方案本身。
